## ✅ 최근 추가 (로컬 개인 일정 + 로컬 알림)

- **일정 추가 기능**: 화면 우하단 **플로팅 버튼(FAB, 현재 월 전례색)** → 편집 시트.
  - 필드: 날짜(필수)·제목(필수)·메모(선택)·시간(종일 또는 `HH:mm`)·알림 토글.
  - **기기 내부(`shared_preferences`)에만 저장** — 클라우드/기기 캘린더 연동 없음.
  - 하단 정보영역 요약 + 상세 "내 일정" 목록(수정/삭제) + 달력 그리드 마커(전례색 점과 구분되는 우상단 점).
- **로컬 알림(서버 없음)**: `flutter_local_notifications` + `timezone`.
  - **전날 21:00 + 당일**(시간 지정 시 그 시각, 종일은 09:00) 예약. 일정별 알림 토글(기본 켜짐).
  - iOS/Android만 동작(웹은 조건부 import 로 알림 없이 나머지 기능 정상).
  - 부정확 예약(inexact) 사용 → Android 정확 알람 특별 권한 불필요. 재부팅/앱 재설치 후 재등록(boot 리시버 + 앱 실행 시 재동기화).
- **일정 = 카테고리 선택 방식** (제목 직접 입력 없음): 사용자가 자기 카테고리를 만들고, 일정 추가 시 **카테고리를 클릭**해 등록. 메모만 부가 설명으로 직접 입력.
  - 카테고리: **이름 + 색상**(아이콘 없음).
  - **"카테고리" 화면**: 일정 편집 시트의 카테고리 칸을 탭 → 전체화면 진입. 기본은 **선택 모드**(항목 탭 → 이전 화면에 자동 설정 후 닫힘, 하단 '카테고리 추가' 버튼).
  - 앱바 **설정(⚙) 버튼** → **편집 모드**(각 항목에 이름/색 수정·삭제·이동 아이콘), 버튼은 **저장**으로 바뀌고 저장 시 현재 상태 일괄 커밋.
  - **첫 실행 시 기본 카테고리 시드**: 본당 행사·교구 행사·기도·전례·교리 (모두 편집/삭제 가능).
  - 카테고리 **이름/색 변경 → 기존 일정에도 반영**. **삭제는 사용 중이면 차단**('사용 중' 표시 + 안내).
- 브랜치: `feat/local-events` (커밋 완료). 검증: `flutter analyze` 클린, `flutter test` 29개 통과, `flutter build web` 성공.

## ✅ 최근 추가 (버그 수정: 화면 회전 잠금 + 위젯 자정 갱신)

- **화면 회전 잠금**: 세로(정방향) + 상하 반전만 지원, 가로(landscape) 미지원.
  - Flutter `SystemChrome.setPreferredOrientations`, Android `screenOrientation="sensorPortrait"`, iOS `Info.plist` 방향 배열(iPhone/iPad 모두 portrait + upside-down).
- **위젯 날짜가 자정 이후에도 전날을 가리키던 버그 수정** (근본 원인: '오늘'을 앱이 스냅샷 저장 시점에 미리 구워넣어, 위젯이 다시 그려져도 전날 데이터를 표시).
  - 이제 위젯이 **현재(엔트리) 날짜로 '오늘'을 직접 판정**하고 스냅샷 42칸 격자에서 해당 날을 찾아 그림(스냅샷 각 날에 전체 제목/날짜 라벨 추가).
  - **iOS**: 향후 8일치 '자정' 타임라인 엔트리 생성(`Calendar.autoupdatingCurrent`, `.atEnd`) → 자정마다 자동 재렌더링. 앱 실행 시 `reloadTimelines`도 유지.
  - **Android**: `AlarmManager`(부정확 예약, 특별 권한 불필요)로 매일 자정 위젯 자체 갱신 예약 + 재부팅/앱 업데이트 후 재예약(부팅 인텐트 필터). 앱 실행 시 재렌더링도 유지.
  - 한계(자연 보정): 달이 바뀌는 자정에 **큰(달력) 위젯**은 다음 앱 실행 전까지 이전 달 격자를 보일 수 있음(작은 위젯은 정상 보정).
