## ★ 이번에 바뀐 점 (CBCK 공식 데이터 통합)

- 공식 매일미사 엔드포인트(`missa.cbck.or.kr/MissaLoad`)에서 **2025~2026년 전례력을 가져와 앱에 동봉**(`assets/calendar/cbck_days.json`, 730일). 이제 해당 기간은 **공식 명칭·전례색·특별 주일(교황 주일 등)·성경 구절 참조·매일미사 링크**를 그대로 사용합니다.
- CBCK 범위 밖(2027년~)은 **계산 엔진이 자동 폴백**(오프라인, 무한 연도).
- **웹 브라우저는 CORS 때문에 이 API를 직접 못 부릅니다.** 그래서 "런타임 호출"이 아니라 "개발 시점 임포트 → 정적 동봉" 방식으로 넣었습니다(웹·모바일 모두 오프라인 동작).

## 1. ⚠️ 당신이 결정/확인해야 할 것 (가장 중요)

- [ ] **⚠️ CBCK 데이터 이용 관련 법무/ToS 판단**: 비공식 내부 엔드포인트에서 데이터를 가져와 앱에 동봉·재배포하는 것이 괜찮은지 **프로젝트 소유자(당신)가 판단/허가 확인** 필요.
  - 성경 "구절 참조"만 저장(본문 아님) → 저작권 안전. 전례일 명칭·날짜는 사실.
  - 그러나 **컴파일된 전례력의 재배포·엔드포인트 이용 약관**은 별개 문제 → 한국천주교주교회의(CBCK)에 **공식 데이터 이용 허가/제휴 문의** 권장. (특히 상용/스토어 배포 시)
- [ ] **앱 이름·아이콘·브랜딩 확정**: 현재 "가톨릭 전례력". 아이콘은 Flutter 기본 → **교체 필요**.
- [ ] **번들 ID 확정**: 현재 임시 `app.catholiccalendar` (스토어 등록 전 확정, 이후 변경 어려움).
- [ ] **웹 호스팅 선택**: Cloudflare Pages / Firebase Hosting / GitHub Pages + 도메인. (`/2026/07` URL 위해 SPA fallback 설정 필요)
- [ ] **주 시작 요일**: 현재 일요일 고정. 월요일 옵션 필요 시 알려주세요.

## 2. 데이터 (CBCK 통합으로 대부분 해결, 나머지만)
