# 스피드다이얼 FAB + 설정 화면 - 설계 (Design)

작성일: 2026-07-23
브랜치: `feature/설정화면-스피드다이얼`

## 배경 / 목표

메인 캘린더 화면의 추가 버튼과 앱 설정을 개선한다.

1. **스피드다이얼 FAB** - 현재 `+` 플로팅 버튼은 탭 시 액션시트/바텀시트(`showAddEventChoice`)로
   "일정 추가 / 축일 추가"를 고른다. 이를 **스피드다이얼**로 바꾼다: `+`를 누르면 스크림이
   덮이고 위로 미니 버튼 3개(① 일정 추가 ② 축일 추가 ③ 설정)가 라벨과 함께 순차로 펼쳐지며,
   `+` 아이콘은 45° 회전해 닫기(X)가 된다. 스크림이나 `+` 재탭 시 닫힌다.
2. **설정 화면(신규)** - 스피드다이얼의 "설정"으로 진입. 백업(iCloud/Google) 관리, 개인정보
   처리방침 링크, 앱 버전 Footer(업데이트 가능 시 스토어 이동)를 제공한다.
3. **백업 전략 개편** - 변경마다 자동 백업하던 방식이 Android에서 Google 로그인 UI를 반복
   노출하는 문제가 있어, **수동 백업 + 10일 주기 알림** 모델로 전환한다.

디자인 시안(제공됨): 설정 화면은 옅은 베이지 배경 + 흰 카드 섹션, 하단 중앙 버전 Footer.
설정 화면에는 **AdMob 배너를 넣지 않는다.**

## 확정된 결정 (사용자 확인)

- 스피드다이얼: **직접 구현**(새 의존성 없이 `Stack` + `AnimationController`), 시안대로.
- 설정 화면 네비게이션: **`Navigator.push(MaterialPageRoute)`** (이 앱은 상세/편집 화면을 이미
  명령형 push로 띄운다. go_router는 월/일 URL만 담당하므로 라우터를 건드리지 않는다).
- 약관/정책: **외부 URL을 인앱 브라우저**로 연다. 이용약관은 없으므로 **개인정보 처리방침만** 표시.
  - 개인정보 처리방침: `https://sidore.org/catholic-calendar-privacy`
- 백업 카드 탭: **별도 백업 화면으로 이동**(상태 + 지금 백업 + 복원 + 설정 안내).
- **백업 전략 변경(중요)**: 변경마다 자동 백업하던 방식을 폐기하고 **수동 백업 + 주기 알림**으로 전환.
  - 이유: Android는 iOS의 iCloud key-value 같은 무음 저장 경로가 없어, 일정/축일 추가·수정·삭제
    때마다 Google Drive 저장이 "로그인 중입니다" 바텀시트를 띄운다(사용자 보고).
  - 자동 백업(변경 시 `backupNow()` 호출)을 **제거**한다. 백업은 오직 사용자가 직접 실행.
  - **마지막 백업 후 10일 경과** 시 앱 실행 때 알림 팝업 → **확인 누르면 백업 화면으로 이동만** 한다
    (팝업이 직접 백업하지 않음).
- 버전 Footer: **업데이트 가능 시에만 탭 → 스토어**. 기존 `AppUpdateService` 재사용.
  강조 문구는 "새로운 기능을 만나보세요 >".

## 기존 코드 재사용 (조사 결과)

- `showAddEventChoice`(`lib/features/events/presentation/add_event_choice.dart`) - 현재 FAB 동작.
  스피드다이얼로 대체되며, 이 함수(및 호출부)는 제거한다.
- `showEventEditor` / `showSaintFeastEditor` - 일정/축일 편집 진입(그대로 재사용).
- `AppUpdateService`(`lib/features/app_update/app_update_service.dart`) - `check()`(GET
  `/app/version?platform=&version=`)로 업데이트 정책 조회, `openStore()`로 스토어 이동, `storeUri()`.
  Footer 업데이트 이동에 재사용.
- 백업: `personalCloudBackupStoreProvider`(`PersonalCloudBackupStore`) + `personalCloudBackupControllerProvider`
  (`PersonalCloudBackupController`).
  - `PersonalCloudBackupStore.checkAvailability()` → `available | notConfigured | unsupported`
  - `PersonalCloudBackupStore.promptSetup()` → Android: Google 로그인/동의, iOS: 설정앱 열기
  - `PersonalCloudBackupController.backupNow()` → 스냅샷 내보내 클라우드 저장(자동 백업과 동일 경로)
  - `PersonalCloudBackupController.restoreIfAvailable()` → 클라우드 스냅샷으로 로컬 복원(전체 교체)
  - (변경) 현재는 이벤트/카테고리 변경마다 `backupNow()`가 자동 호출된다. **이 자동 호출을 제거**하고
    수동 백업으로 전환한다(아래 백업 전략 참고).
- `backup_notice.dart`(`maybeShowBackupNotice`) - 첫 일정 추가 시 1회 표시되는 백업 안내/설정 유도.
  현재 문구가 "자동으로 저장"이라 수동 백업 모델에 맞게 수정 필요.
- `package_info_plus` - 이미 의존성 있음(`app_update_service`, `category_log_service`에서 사용).
- `url_launcher` - 인앱 브라우저(`LaunchMode.inAppBrowserView`) → 실패 시 `externalApplication` 폴백
  패턴이 `day_detail_view.dart`/`saint_feast_editor_page.dart`에 있음. 개인정보 링크에 동일 적용.
- FAB 색 = 현재 월 전례색: `seasonColor(mid.season)` + `context.liturgical.of(...)` (기존 `_addEventFab`).

## 구성요소 설계

### (A) 스피드다이얼 FAB - `SpeedDialFab`

- 신규 위젯. `calendar_page.dart`의 `_addEventFab`를 대체하고, `_narrow`/`_wide`의 `Positioned`
  위치에 그대로 배치.
- 입력: `color`(전례색), `onAddEvent`, `onAddFeast`, `onOpenSettings` 콜백(또는 `date`/`ref`를
  받아 내부에서 호출). 상태: `AnimationController`(open 0→1).
- 열림 UI:
  - 반투명 스크림(`ModalBarrier` 또는 `GestureDetector` + 반투명 `Container`), 탭 시 닫힘.
  - 미니 FAB 3개가 아래→위로 stagger 등장(각 항목 `Interval`로 지연), 각 왼쪽에 검정 라벨 pill.
    - ① 일정 추가 `Icons.event` → `onAddEvent`
    - ② 축일 추가 `Icons.star_border` → `onAddFeast`
    - ③ 설정 `Icons.settings` → `onOpenSettings`
  - 메인 버튼 `+` 아이콘 45° 회전(→ X). 미니 버튼은 흰 배경, 아이콘은 전례색 계열.
- 닫힘: 스크림 탭 / `+`(X) 재탭 / 항목 선택 후. 항목 선택 시 먼저 닫고 해당 액션 실행.
- 접근성: 각 미니 버튼에 `tooltip`/`Semantics` 라벨.

### (B) 설정 화면 - `SettingsPage`

- `Scaffold`(배경 = 시안의 옅은 베이지) + `AppBar`(`< 설정`, `centerTitle: true`). **AdMob 없음.**
- 본문: 섹션 헤더(회색 소문자 라벨) + 흰 카드 리스트.
  - **백업** 섹션(카드 1개):
    - iOS: `Icons.cloud_upload_outlined`(초록) · 제목 "iCloud 백업" · 부제 "iCloud에 일정을 백업하고 복원합니다"
    - Android: 제목 "Google 백업" · 부제 "Google Drive에 일정을 백업하고 복원합니다"
    - (수동 백업 모델이므로 "자동" 표현은 쓰지 않는다. 시안의 "자동 백업합니다" 문구는 이 방침에 맞게 대체.)
    - 우측 chevron → (C-2) `BackupSettingsPage`로 push
    - 플랫폼이 `unsupported`(웹/데스크톱)이면 이 섹션 자체를 숨김
  - **약관 및 정책** 섹션(카드 1개):
    - "개인정보 처리방침" `Icons.shield_outlined` → 개인정보 URL을 인앱 브라우저로 열기
      (실패 시 external 폴백). 이용약관 항목 없음.
- 하단: (D) 버전 Footer.

### (C) 백업 전략: 수동 백업 + 10일 주기 알림

자동 백업(변경마다 저장)을 폐기하고 아래로 대체한다. 이 변경이 이 기능의 핵심 중 하나다.

**자동 백업 제거**
- `EventStore`의 변경 메서드(`add`/`update`/`delete` 등, `event_providers.dart` ~line 181)와
  `CategoryStore`의 변경(`category_providers.dart` ~line 77)에서 `backupNow()` 호출을 제거한다.
- 이로써 Android의 "로그인 중입니다" 바텀시트가 변경 때마다 뜨던 문제가 사라진다.
- iOS도 동일하게 수동 모델로 통일한다(무음 iCloud라 문제는 없었지만 동작 일관성 위해).

**마지막 백업 시각 저장**
- SharedPreferences 키 `personal_backup_last_at`(ISO8601 UTC). `backupNow()` 성공 시 갱신.
- 백업 화면에 "마지막 백업: N일 전 / 아직 없음" 표시.

**10일 주기 알림 팝업**
- 앱 콜드 스타트 시(iOS/Android만, 웹 제외) 기존 업데이트 체크/복원 흐름 뒤에 평가한다.
- 조건: 사용자에게 보호할 개인 데이터가 있고(구현 기준: **이벤트가 있음**. 시드/사용자
  카테고리 구분이 모호하고 카테고리만 있고 이벤트가 없는 경우는 잃을 데이터가 적어 이벤트 유무로 판정)
  `now - baseline >= 10일`.
  - `baseline` = `personal_backup_last_at`와 `personal_backup_reminder_at` 중 **더 최근 시각**
    (둘 다 없으면 최초로 간주). 마지막 백업이 오래됐어도 방금 알림을 띄웠으면(reminderAt이 더
    최근) 스누즈가 유지되어, 백업 이력이 있는 사용자에게도 실행마다 반복 노출되지 않는다.
  - `personal_backup_reminder_at`: 알림을 평가/표시한 시각을 저장하는 키. 데이터가 있는데 아직
    백업/알림 이력이 없으면 이번 실행 시각으로 시드 → 첫 알림은 사용 시작 후 약 10일에 뜬다.
- 팝업: 제목 "백업이 필요해요"(예시), 본문 "마지막 백업 후 10일이 지났어요. 지금 백업할까요?".
  - **확인** → 백업 화면(`BackupSettingsPage`)으로 이동(백업은 화면에서 사용자가 직접).
  - **나중에**/닫기 → 그냥 닫음.
- 표시 후 동작과 무관하게 `personal_backup_reminder_at = now`로 갱신 → 최소 10일 간격으로만 재노출.
- 최대 노출 빈도: 앱 실행당 1회.

**첫 일정 추가 안내(`backup_notice.dart`) 조정**
- 초기 설정 유도용으로 유지하되 "자동으로 저장" 문구를 수동 모델에 맞게 수정
  ("설정 > 백업에서 언제든 백업/복원할 수 있어요" 취지).
- 사용자가 이 안내에서 설정을 완료하면 초기 1회 `backupNow()` 실행 + `personal_backup_last_at` 갱신.

### (C-2) 백업 서브화면 - `BackupSettingsPage`

- 진입 시 `checkAvailability()` 호출(로딩 표시) → 상태에 따라:
  - `available`: "연결됨" 상태 카드 + "마지막 백업: …" 표시.
  - `notConfigured`: "백업 설정이 필요합니다" + [설정하기] → `promptSetup()` 후 상태 재조회.
  - `unsupported`: 설정 화면에서 진입 자체가 막히므로 정상 흐름에선 도달하지 않음(방어적으로 안내만).
- 액션:
  - **지금 백업** → `backupNow()` → 성공 시 `personal_backup_last_at` 갱신 + 완료 스낵바(실패 시 실패 스낵바).
    (사용자가 직접 누른 것이므로 Android에서 로그인/권한 UI가 떠도 정상.)
  - **복원** → 확인 다이얼로그("현재 기기의 일정/카테고리를 클라우드 백업으로 **덮어씁니다**(전체 교체).
    계속할까요?") → 확인 시 `restoreIfAvailable()` → 결과 스낵바(복원됨/백업 없음/실패).
- 이 화면은 상태 확인 + 수동 백업/복원 + 설정 안내 용도. 자동 백업 토글 없음(자동 백업 자체가 없음).

### (D) 버전 Footer (설정 화면 하단)

- `packageInfoProvider`(신규 `FutureProvider<PackageInfo>`)로 버전/빌드 조회.
- 표시: "가톨릭 달력" + "버전 {version} (빌드 {buildNumber})" 중앙, 회색.
- 업데이트 체크: 설정 화면 진입 시 `AppUpdateService().check()` 1회.
  - 정책 반환(=업데이트 가능): Footer가 **탭 가능** + "새로운 기능을 만나보세요 >" 강조 문구 → 탭 시 `openStore()`.
  - null(최신/실패/오프라인) 또는 웹: 버전만 표시, 탭 없음.

### (E) 파일 구조 / 상태관리

- 신규 `lib/features/settings/presentation/`:
  - `settings_page.dart` - `SettingsPage`
  - `backup_settings_page.dart` - `BackupSettingsPage`
- 스피드다이얼 위젯: `lib/features/calendar/presentation/widgets/speed_dial_fab.dart`
  (캘린더 화면 전용 위젯이므로 calendar widgets 하위).
- 신규 provider: `packageInfoProvider`(`FutureProvider<PackageInfo>`) - 위치는
  `lib/features/app_update/app_update_providers.dart`(신규). 버전 표시와 업데이트 체크가 함께
  묶이므로 app_update 하위에 둔다. 백업 provider는 기존 것 재사용.
- 개인정보 URL은 `settings` 상수(`const kPrivacyPolicyUrl = 'https://sidore.org/catholic-calendar-privacy';`).
- `calendar_page.dart`: `_addEventFab` → `SpeedDialFab`로 교체, `showAddEventChoice` import/호출 제거.
- `add_event_choice.dart`: 더 이상 쓰이지 않으면 삭제(다른 참조 없을 때).
- 백업 전략 변경 관련 수정:
  - `event_providers.dart` / `category_providers.dart`: 변경 시 `backupNow()` 호출 제거.
  - `backupNow()` 성공 시 `personal_backup_last_at` 저장(컨트롤러 또는 백업 화면에서).
  - `backup_notice.dart`: 문구 수정 + 설정 완료 시 초기 백업/타임스탬프 갱신.
  - 10일 알림: `lib/features/events/presentation/`(또는 settings) 헬퍼 `maybeShowBackupReminder(context, ref)`
    신설, `app.dart` 콜드 스타트 흐름에서 호출. 타임스탬프 키는 `backup_notice.dart` 인근에 상수화.

### (F) 에러 처리

- 백업/복원 실패, 설정 미완료: 스낵바/상태 카드로 사용자에게 안내(기존 컨트롤러가 예외를 삼키고
  false/no-op 반환하므로 반환값으로 분기).
- 업데이트 체크 실패/타임아웃: 조용히 무시(버전만 표시). 기존 `check()`가 예외 시 null 반환.
- 인앱 브라우저 실패: external 폴백. 그것도 실패하면 조용히 무시(로그).

### (G) 테스트

- 위젯 테스트:
  - 스피드다이얼: 초기엔 미니 버튼 미표시 → `+` 탭 시 3개(일정/축일/설정) 표시, 스크림 표시,
    스크림 탭 시 닫힘. 각 항목 탭 시 콜백 호출.
  - 설정 화면: 백업/약관 섹션·항목 렌더, 플랫폼별 라벨(iCloud vs Google), 개인정보 항목 존재.
  - 백업 화면: `notConfigured`일 때 [설정하기] 노출, `available`일 때 지금 백업/복원 노출.
  - Footer: 업데이트 정책 있음 → 탭 가능/문구 노출, 없음 → 버전만.
  - 백업 전략: 이벤트/카테고리 변경 시 `backupNow()`가 **호출되지 않음** 검증(fake 컨트롤러 호출 0회).
    10일 알림: `personal_backup_last_at`가 10일 초과 + 데이터 있음 → 팝업 노출·확인 시 백업 화면 이동,
    10일 이내면 미노출. (SharedPreferences mock으로 시각 주입.)
- 주입: `AppUpdateService`는 mock `http.Client`, 백업은 `PersonalCloudBackupStore`/컨트롤러를
  fake로 override(Riverpod `overrides`). `PackageInfo.setMockInitialValues`로 버전 고정.
- 검증: `flutter analyze` 클린, `flutter test` 통과.
